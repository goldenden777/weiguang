# 模拟环境运行说明

在**无后端、无数据库**的情况下，用前端 Mock 数据在本地完整跑通系统，用于验证功能正确性。

---

## 一、启动模拟环境

### 方式一：双击启动（推荐）

1. 在**项目根目录**（`线下活动系统` 文件夹内）双击 **`启动模拟环境.bat`**。
2. 首次运行会自动安装依赖并启动开发服务器；若提示“未找到 npm”，请先安装 Node.js（见下方「未找到 npm 时」）。
3. 出现 `Local: http://localhost:4321` 后，在浏览器打开 **http://localhost:4321** 即可进行模拟测试。

也可在 **`js/code`** 目录下双击 **`安装并运行.bat`**，效果相同。

### 方式二：在终端中执行

在项目根目录下打开终端，执行：

```bash
cd js/code
npm install
npm run simulate
```

或在 **`js/code`** 目录下直接执行 `npm install` 与 `npm run simulate`。

浏览器访问终端中显示的地址（通常为 **http://localhost:4321**）。

### 未找到 npm 时

若终端或脚本提示“无法将 npm 项识别为……”或“未找到 npm”：

1. **安装 Node.js**：打开 https://nodejs.org/ ，下载并安装 **LTS 版本**，安装时**勾选 “Add to PATH”**。
2. **安装完成后**：关闭所有终端和 Cursor 窗口，再重新打开项目，然后再次双击 **`启动模拟环境.bat`** 或在终端执行上述命令。

若已安装 Node 但终端仍找不到，可在 **PowerShell** 中先用完整路径试一次（Node 默认安装路径）：

```powershell
cd "D:\基金会\线下活动系统\js\code"
& "C:\Program Files\nodejs\npm.cmd" install
& "C:\Program Files\nodejs\npm.cmd" run simulate
```

---

- **模拟环境**：所有活动、报名、用户、配置等数据均来自前端 `src/data/` 与各 `*.service.ts` 中的 Mock，不请求任何后端接口。
- 登录态保存在浏览器 **sessionStorage**（键 `currentUser`），关闭标签页后失效。

## 四、与真实环境的区别

| 项目     | 模拟环境（当前）     | 真实环境（对接后端后）     |
|----------|----------------------|----------------------------|
| 数据来源 | 前端 `src/data/` Mock | 后端 API                   |
| 登录     | 仅校验 Mock 账号密码 | 调用 `/api/auth/login` 等  |
| 报名/活动 | 仅写内存，刷新可恢复 | 持久化到数据库             |
| 部署     | 本地 `npm run dev`   | 前端静态部署 + 后端服务    |

对接真实 API 时，只需在 `src/services/` 下将各 service 的 Mock 实现替换为 `fetch`/`axios` 调用，并参考 **`API_CONTRACT.md`**，无需改页面与组件接口。

## 五、常用命令汇总

```bash
cd js/code
npm install          # 安装依赖
npm run simulate     # 启动模拟环境（同 npm run dev）
npm run build        # 构建静态产物到 dist/
npm run preview      # 本地预览 dist/ 构建结果
```

按上述流程在模拟环境下跑通门户、管理端、机构端，即可验证系统功能正确性。
